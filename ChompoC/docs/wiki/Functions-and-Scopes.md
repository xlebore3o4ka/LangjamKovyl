# Функции и области видимости

## Объявление и вызов

```javascript
fun multiply(left, right) {
    return left * right;
}

var result = multiply(6, 7);
```

Количество аргументов пользовательской функции должно точно совпадать с количеством параметров. Native-функции могут принимать диапазон аргументов.

## Функции первого класса

```javascript
fun add(a, b) {
    return a + b;
}

var operation = add;
print(operation(2, 3));
```

Функцию можно присвоить переменной, передать аргументом и вернуть из другой функции.

## Замыкания

```javascript
fun makeCounter(start) {
    var value = start;

    fun next() {
        value++;
        return value;
    }

    return next;
}

var counter = makeCounter(10);
print(counter(), " ", counter()); // 11 12
```

Closure удерживает лексическое окружение объявления. Захваченная переменная продолжает существовать после завершения внешней функции.

## Лексические scope

```javascript
var value = 1;

{
    var value = 2;
    print(value); // 2
}

print(value); // 1
```

Имя ищется по лексическим областям видимости. Повторное объявление в одном scope запрещено, shadowing во вложенном scope разрешён.

Resolver до исполнения переводит локальное имя в адрес `(depth, slot)`. Это является оптимизацией и не меняет семантику scope. Блок без собственных локальных объявлений может не создавать отдельный runtime-объект `Environment`, но остаётся лексическим блоком языка.

## Рекурсия

```javascript
fun factorial(value) {
    if (value <= 1)
        return 1;

    return value * factorial(value - 1);
}
```

Локально объявленные функции также могут рекурсивно обращаться к себе.

Глубина вызовов ограничена `ChompoConfig::MaxCallDepth` — сейчас 512. Превышение вызывает контролируемую ошибку `Runtime StackOverflow`.

## `return`

```javascript
return value;
return;
```

`return` разрешён только внутри функции. Пустой `return` и естественное завершение функции возвращают `NULL`.

## Граница функции и циклы

`break` и `continue` не переходят через вызов функции. Функция, объявленная внутри цикла, не получает доступ к loop-контексту внешней функции.

## Переиспользование окружений

Runtime переиспользует block/function environments, когда они не захвачены closure. Захваченное окружение не очищается и не выдаётся другому вызову, поэтому оптимизация сохраняет корректность замыканий.

# СL++ (.clx)

CL ++ - это траспилирумый функциональный динамически-типизированный язык в Erlang заточенный специально под AtomVM
Цель языка - Упростить разработку под микроконтроллеры приводя более удобный
и привычный синтаксис для большинства программистов

## Особенности

- Входной точкой программы всегда выступает публичная функция start

## Hello World

```clx
pub fn main() {
    var greeting = "Hello, AtomVM!"
    print(greeting)
}
```

## Типы Данных

number - целые числа и числа с плавающей точкой
bool - булево значение (true/false)
atom - атомы (начинаются с : и маленькой буквы. Пример - :atom)
string - строки (начинаются с " и заканчиваются на "). Пример - "Hello". Являются массивами)
array - массивы (начинаются с [ и заканчиваются на ]). Пример - [1, 2, 3]
tuple - кортежи (начинаются с { и заканчиваются на }). Пример - {1, 2, 3}

## Синтаксис

```clx
# комментарии

# объявление функции
fn foo() {}

# вызов функции
foo()

# анонимная функция
fn () {}

# импортирование модулей (все модули должны лежать в одном каталоге)
import "<module>"

# публичная функция (может быть вызвана из других модулей)
pub fn foo() {}

# возвращение значения
return 

# объявление переменной
var foo = 52

# while цикл
while (foo < 10) {}

# условия
if (foo < 10) {

} else if (foo < 20) {

} else {

}

# итератор по массивам
for (i in [1, 2, 3]) {
    print(i)
}

# операторы сравнения

foo == bar
foo ~= bar
foo < bar
foo > bar
foo <= bar
foo >= bar

# логические операторы
foo and bar
foo or bar

# получение элементов (индексация начиная с 1. Работает для массивов и кортежей)
arr[1]
tup[2]

# математические операторы
foo + bar
foo - bar
foo ++ bar (Конкатенация строк)

# error handling (ставится пред опасной операцией возвращает кортеж (:ok, Result)
# если всё прошло успешно, (:error, Error) если произошла ошибка)
var handle_error = try <some_operation>

# паттерн матчинг меж процессных сообщений
receive {
    (:ok, data) -> {}
    (:error, error) -> {}
}

```

## Чат

Для демонстрации возможностей языка был написан чат (avm файл можно сказать в релизах)

Описание чата:

- Консольный интерфейс
- Возможность регистрации и входа
- Возможность обмена личными сообщениями
- Возможность обмена сообщениями в группе
- Возможность создавать группы
- Управление через простые комманды (/help, /add, /exit, /kick)
- Real-time обновление

Зависимости:

У чата нет внешних зависимостей кроме redis сервера запущенного на порту 6379 на localhost
Это можно изменить в исходниках которые находятся в этом же репозитории

## Самостоятельная компиляция

Зависимости:

- node.js - для сборки
- npm - для управления зависимостями
- erlang - для компиляции erlang файлов
- atomvm - для запуска
- packbem - для упаковки avm файлов


```bash
# ... тут вы копируете репозиторий и переходите в папку проекта

pnpm install

# после этого вы можете компилироват .clx файлы в erlang с помощью следующей команды:
pnpm compile <путь к .clx файлу>
```

## Самостоятельная сборка erlang файлов под atomvm

```bash
# Получаем erlang файлы
pnpm compile <путь к .clx файлу>

# Компилируем erlang файлы в beam
erlc <путь к .erl файлу>

# Если файлов много или нужны внешние зависимости
packbeam <имя пакета> <пути к .beam и .avm файлам>

# Запускаем atomvm
atomvm <имя пакета>
```

### Пример сборки чата

```bash
pnpm compile ./projects/chat/main.clx

erlc ... # компилируем erlang файлы

erlc clx_std.erl # Обязательно надо собрать базовую библиотеку языка

# Пакуем beam файлы в avm
packbeam app.avm <...>.beam clx_std.beam /usr/local/lib/estdlib.avm /usr/local/lib/eavmlib.avm

# Запускаем atomvm
atomvm app.avm

# Увидим сообщение в консоли
Server start on port 8080
```
